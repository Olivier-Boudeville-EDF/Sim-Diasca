get_process_specific_value/0

get_process_specific_value() -> pos_integer()

Returns a value (a strictly positive integer) expected to be as much as possible specific to the current (Erlang) process.

Mostly based on its PID.

Useful for example when a large number of similar processes try to access to the same resource (ex: a set of file descriptors) at the same time: they can rely on some random waiting based on that process-specific value in order to smooth the accesses over time.

We could imagine taking into account as well the current time, the process reductions, etc. or generating a reference.
