start/1

start(Module::module() | [module()]) -> void()

Starts a test; expected to be the first test statement.

Here we disable explicitly the trapping of EXIT events, as a function run through "erl -eval" (like our tests) or through "erl -run" will be executed in a process which will silently trap EXIT events, which would mean that the crash of any process created from the test, even thanks to spawn_link, would most probably remain unnoticed (just leading to an EXIT message happily sitting in the mailbox of the test process).
