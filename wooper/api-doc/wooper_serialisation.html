Description

Module containing facilities for the serialisation and deserialisation of WOOPER instances.

Note: offseting as much as possible code from the counterpart class header in this module allows for smaller, cache-friendly BEAMs, and short compilation times.

deserialise/4

deserialise(BinSerialisation::bin_serialisation(), EntryTransformer::entry_transformer(), UserData::basic_utils:user_data(), ListenerPid) -> any()

  • ListenerPid = maybe(pid())

Deserialises the specified instance from its serialised form (as a term, not as a binary), to obtain its corresponding state, using specified entry transformer and user data, then having the current executing process embody this instance from then on.

Does not return, as the WOOPER main loop will manage from then this just deserialised instance.

(helper, as the receiver process may not even be already a WOOPER instance)

Note: the hosting process is not created here, as, for an increased parallelism, we expect deserialisations to happen directly from the final instance processes; we consider here that the process executing this helper is the final host.

remote_synchronisable_load_link/2

remote_synchronisable_load_link(Node::net_utils:node_name(), BinSerialisation::bin_serialisation()) -> instance_pid()

Spawns on specified node a new instance of this class, based on the specified serialisation information and links it to the current process.

Returns the PID of the created instance.

Creation is asynchronous (the PID is directly returned), however a { spawn_successful, SpawnedPid } message will be received once (if ever) the instance is up and running. This allows to perform the actual instance creations in parallel, by waiting bulks of creations.
