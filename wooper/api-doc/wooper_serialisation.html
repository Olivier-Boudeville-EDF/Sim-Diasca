Description

Module containing facilities for the serialisation and deserialisation of WOOPER instances.

Offsetting as much as possible code from the counterpart class header in this module allows for smaller, cache-friendly BEAMs, and shorter compilation times.

handle_private_processes/2

handle_private_processes(PrivateAttributeNames::[attribute_name()], State::wooper:state()) -> wooper:state()

Replaces any PID value associated to any of the specified attribute names with by 'undefined' atom.

Handles private processes (through the name of the specified attributes), which are processes that are internal to an instance that is to be serialised, so that any next serialisation will see instead of their (former) PID a serialisation marker.

Returns an updated state.

Typically used by any onPreSerialisation/2 overridden request, to avoid that PIDs remain when serialising.

load_instances/1

load_instances(Serialisation::serialisation()) -> [instance_pid()]

Loads the instances stored in the specified serialisation, using no specific entry transformer or user data: spawns them and returns a list of their corresponding PIDs, in no specific order (as usually it does not matter).

These creations will be asynchronous: this function returns legit PIDs as soon as the creations are triggered, without waiting for them to complete.

Created instances will not be linked to the calling process.

Loads the instances stored in the specified serialisation, using the specified entry transformer and user data: spawns them and returns a list of their corresponding PIDs, in no specific order (as usually it does not matter).

Note that all these deserialisations will use the specified user data, and that any updated version of it they would return will be ignored.

Created instances will not be linked to the calling process.
